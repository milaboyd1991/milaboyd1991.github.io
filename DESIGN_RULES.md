# Mila Boyd Portfolio — Design Rules

These are the permanent design standards for this college portfolio website
(`C:\Users\Mila\Desktop\money`). Read this file **before editing any page** and
apply it as the governing standard. Reuse the existing `css/portfolio.css` design
system; do not invent one-off styles unless truly necessary.

---

## 1. Strategy First
Every page must support both user value and professional goals. The site should help
visitors quickly understand who I am, what I design, what skills I have, and how to
contact me.

## 2. Product Design Mindset
Design is not only visuals. Every page should show: problem solving, clear goals,
research thinking, strategy, prototyping, testing, and launch-ready presentation.

## 3. Great UX
The portfolio must be: usable, clear, delightful, trustworthy, emotionally calm,
easy to scan, accessible, and professional.

## 4. Wireframe Before Polish
Before changing a page, identify the structure first — hero, intro, content sections,
project cards, CTA, footer. Do not decorate before the layout makes sense.

## 5. Design System Consistency
Use the existing `portfolio.css` system whenever possible. Keep consistent: colors,
typography, spacing, buttons, cards, nav, and section rhythm. Do not create random
one-off styles unless necessary.

## 6. Design Thinking
Apply Empathize → Define → Ideate → Prototype → Test → Implement. For the portfolio,
every project should clearly explain: the problem, the goal, the process, the
solution, and the result.

## 7. Gestalt Principles
- **Proximity** — related content stays grouped.
- **Similarity** — cards, buttons, and headings look consistent.
- **Continuity** — the eye should flow smoothly down the page.
- **Closure** — layouts should feel complete, not unfinished.
- **Figure-ground** — text must stand clearly against backgrounds.
- **Common region** — sections should be visually grouped.
- **Repetition** — repeat patterns for trust and polish.

## 8. Grid + Responsive Design
Use clean, responsive layout: mobile-first, Bootstrap grid where appropriate,
consistent containers, max-width readable content, strong alignment, no
floating/random placement. Project cards must stack cleanly on mobile.

## 9. Cognitive Load
Reduce mental effort: remove clutter, filler, and duplicate content; use short
sections, clear labels, and progressive disclosure; keep one primary CTA per section;
do not overwhelm the visitor.

## 10. Accessibility
Every page must include: one clear `h1`; logical `h2`/`h3` order; readable contrast;
descriptive `alt` text; accessible buttons and links; `aria-current="page"` on the
active nav item; and a responsive mobile layout.

## 11. Portfolio Quality
The website should feel like a modern UX/UI portfolio, a digital-design-student
portfolio, a professional creative designer, and a polished, job-application-ready
presentation. It should **not** feel like an old Dreamweaver template, a cluttered
class-project dump, a random business website, a generic Bootstrap starter, or a
childish/messy layout.

## 12. Page Cleanup Rule
When cleaning any page:
1. Read the file first.
2. Identify what is broken, outdated, cluttered, or inconsistent.
3. Reuse the site's existing design system.
4. Fix hierarchy, spacing, buttons, cards, and headings.
5. Do not touch unrelated mini-sites (e.g., `PNW-Portfolio-Kit/`).
6. Do not add credentials, private data, passwords, or server info.
7. Verify links and images after editing.

---

*Apply these rules as the standard before rebuilding any page, including `portfolio.html`.*
